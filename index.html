微信分享在 iOS SDK 1.8.6.1 版本与之前版本的区别

发表于 2020-06-18 | 分类于 react-native 花里胡哨的实践

微信 SDK 在 1.8.6.1 版本之后，iOS 上改为通过 Universal Links 跳转完成应用安全校验。在做微信分享时主要注意的几个不同的地方。Universal Links 的设置网上有很多文章，按步骤来基本没有踩坑

  1. xcode info 标签中的 LSApplicationQueriesSchemes 多添加了一个 weixinULAPI；
  2. WXApi registerApp 方法需要多传一个 universalLink 参数；
  3. WXApi sendReq 方法需要多传一个 completion 回调函数，可以是 nil；

  4. 一定要重写 AppDelegate 的 continueUserActivity 方法，官方文档用红色文字标识

js 中 0 字符串是 true 还是 false

发表于 2019-02-12 | 分类于 基础知识不牢踩的坑

今天做项目的时候发现后台返回的值是 0 字符串，用这个值来做 if 判断，结果居然通过了。于是做了如下测试，结果一脸蒙圈

于是翻了一下基础知识。

  1. 首先来看 if(a) 这个判断。在 if 语句里面，判断条件会自动条用 Boolean() 转换函数将判断条件转换成一个布尔值。也就是说 0 字符串调用 Boolean() 的结果是 true 。为什么呢？我们来看 Boolean() 的转换规则：
数据类型 转换为 true 的值 转换为 false 的值
Boolean true false
String 任何非空字符串 空字符串 “”
Number 任何非零数值 0 和 NaN
Object 任何对象 null
Undefined n/a (不适用) undefined

0 字符串属于非空字符串，结果自然就是 true；

  1. 再看 if(a == false) 的情况；这就要看 == 运算符的比较规则了。相等运算符，在比较时会先转换操作数，然后在比较他们的相等性，转换的基本规则如下：
    1) 如果一个操作数是布尔值，先将其转换为数值；
    2) 如果一个操作数是字符串，另一个是数值，先将其转换为数值；
    3) 如果一个操作数是对象，另一个不是，先调用 valueOf() 方法得到基本类型，再进行比较；

    比较时遵循以下原则：
    1) null 和 undefined 相等；
    2) 如果有一个操作数是NaN，则返回false；
    3) 如果两个操作数都是对象，看他们是否指向同一个对象；
    4）在比较之前，不能将 null 和 undefined 转换成其他任何值。

在这个例子中，a 是字符串，false 是布尔值，都需要转换成数值，转换后都是 0，所以结果也是 true；
